# Refleksjon – HelpdeskGuard

## Hva fungerte bra
Jeg synes strukturen på prosjektet fungerte bra. Ved å dele opp koden i flere filer som AuthStore, TicketStore og egne Views, ble det lettere å jobbe med prosjektet.

SwiftUI gjorde det enkelt å lage brukergrensesnitt, og Git hjalp meg med å holde oversikt over endringer.

Jeg fikk også til å sette opp en server med Ubuntu, Docker og Uptime Kuma, noe som ga meg bedre forståelse av drift.

## Hva var utfordrende
Det vanskeligste var:
- autentisering (innlogging og lagring av brukere)
- Git merge konflikter
- kobling mellom UI og data

Jeg brukte tid på å forstå hvordan data skulle lagres og vises riktig.

## Hva jeg ville gjort annerledes
Hvis jeg startet på nytt:
- jeg ville planlagt datastrukturen bedre først
- jeg ville implementert sikker lagring tidligere
- jeg ville laget backend tidligere i prosjektet

## Hva jeg har lært
- hvordan SwiftUI fungerer i praksis
- hvordan Git brukes i et prosjekt
- grunnleggende serverdrift (Linux, Docker, sikkerhet)
- viktigheten av dokumentasjon

## Videre utvikling
- lage backend (FastAPI)
- implementere sikker innlogging (hashing + JWT)
- vise liste over saker i appen
- forbedre UI og brukeropplevelse
